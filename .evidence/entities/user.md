# User｜用户

> 系统使用者。用户本身只保存身份信息，工作空间访问通过 `UserWorkspaces` 和 `Member` 建立。

## 基本信息

| 项目     | 值                                     |
| -------- | -------------------------------------- |
| 模型类型 | Aggregate                              |
| 领域类型 | Participant / Party                    |
| 主标识   | `id`                                   |
| 主要职责 | 查询用户信息、访问该用户的工作空间集合 |

## 属性

| 属性    | 类型      | 必填 | 说明                |
| ------- | --------- | ---- | ------------------- |
| `id`    | `string`  | 是   | 用户 ID，非自增主键 |
| `name`  | `string`  | 是   | 用户名称            |
| `email` | `string?` | 否   | 邮箱                |

## 行为

| 操作                       | 说明                         |
| -------------------------- | ---------------------------- |
| `find_by_identity(userId)` | 按 ID 查找用户               |
| `workspaces()`             | 返回用户可访问的工作空间集合 |

## 关联

| 关系     | 目标                      | 说明                                 |
| -------- | ------------------------- | ------------------------------------ |
| 访问     | [Workspace](workspace.md) | 通过 `UserWorkspaces` 查询           |
| 成员身份 | [Member](member.md)       | 用户在工作空间中的角色由 Member 表示 |
